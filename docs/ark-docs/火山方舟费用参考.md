# 影片生成模型 (Seedance) 計費說明

影片生成模型採用**非同步推理**，按輸出影片消耗的 token 量進行計費（元/百萬 token）。

## 1. 按 Token 單價表



| **模型名稱**                     | **線上推理 (元/百萬token)**                        | **離線推理 (元/百萬token)** |
| -------------------------------- | -------------------------------------------------- | --------------------------- |
| **doubao-seedance-2.0**          | - 輸入不含影片：46.00 - 輸入包含影片：28.00        | 暫不支援                    |
| **doubao-seedance-2.0-fast**     | - 輸入不含影片：37.00 - 輸入包含影片：22.00        | 暫不支援                    |
| **doubao-seedance-1.5-pro**      | - 有聲：16.00 - 無聲：8.00                         | - 有聲：8.00 - 無聲：4.00   |
| **doubao-seedance-1.0-pro**      | 15.00                                              | 7.50                        |
| **doubao-seedance-1.0-pro-fast** | 4.20                                               | 2.10                        |
| **doubao-seedance-1-0-lite**     | 10.00                                              | 5.00                        |

### 計費規則與 Token 估算說明：

- **計費條件**：僅對成功生成的影片計費。因稽核等原因導致生成失敗的，不收取費用。
- **影片價格估算公式**：`按 token 單價 × token 用量`
- **Seedance 2.0 / 2.0-fast token 估算**：`(輸入影片時長 + 輸出影片時長) × 寬 × 高 × 幀率 / 1024`
  - *注：輸入包含影片時，存在最低 token 用量限制（見下方表格），若估算用量 < 最低限制，則按最低用量計費。*
- **Seedance 1.x token 估算**：`(寬 × 高 × 幀率 × 時長) / 1024`
- **Draft 影片（僅480p）估算公式**：`正常影片 token 用量 × 折算係數`
  - *注：`doubao-seedance-1.5-pro` 的 token 折算係數為：無聲 0.7；有聲 0.6。*
- **準確用量**：以呼叫 API 後返回資訊中的 `usage` 欄位為準。

# 圖片生成模型 (Seedream) 計費說明

圖片生成模型按**成功輸出的圖片數量**（元/張）進行計費。

## 1. 按張數單價表

| **模型名稱**                 | **單價 (元/張)** |
| ---------------------------- | ---------------- |
| **doubao-seedream-5.0-lite** | 0.22             |
| **doubao-seedream-5.0**      | 0.22             |
| **doubao-seedream-4.5**      | 0.25             |
| **doubao-seedream-4.0**      | 0.20             |
| **doubao-seedream-3.0-t2i**  | 0.259            |

### 計費規則說明：

- **計費條件**：按成功輸出的圖片數量計費。因稽核等原因未成功輸出的圖片不計費。
- **組圖場景**：按實際生成的圖片數量計費。

# 大語言模型 (文字生成) 計費說明

大語言模型按 token 後付費，費用公式：`輸入單價 × 輸入 token + 輸出單價 × 輸出 token`。

部分模型按輸入長度分段計費，不同區間 token 單價不同。
