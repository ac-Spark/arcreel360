# 完整入門教程

本教程指導你從零開始，使用 ArcReel 將小說轉換為短影片。

## 你將學到

1. **環境準備** — 獲取 API 金鑰
2. **部署服務** — 透過 Docker 部署
3. **完整流程** — 從小說到影片的每一步操作
4. **進階技巧** — 重新生成、費用控制、本地開發

## 預計耗時

- 環境準備：10-20 分鐘（僅首次需要）
- 生成一個 1 分鐘影片：約 30 分鐘

## 第一章：環境準備

### 1.1 獲取圖片/影片生成供應商 API 金鑰

ArcReel 支援多個供應商，**至少配置一個**即可開始使用：

| 供應商 | 獲取地址 | 說明 |
|--------|---------|------|
| **Gemini** (Google) | [AI Studio](https://aistudio.google.com/apikey) | 需付費層級，新使用者自動獲 $300 贈金 |
| **火山方舟** | [火山引擎控制檯](https://console.volcengine.com/ark) | 按 token/張數計費 (CNY) |
| **Grok** (xAI) | [xAI Console](https://console.x.ai/) | 按張/秒計費 (USD) |
| **OpenAI** | [OpenAI Platform](https://platform.openai.com/) | 按張/秒計費 (USD) |

也可以在部署後透過設定頁新增**自定義供應商**（任何 OpenAI 相容 / Google 相容 API）。

> ⚠️ API 金鑰是敏感資訊，請妥善保管，不要分享給他人或上傳到公開倉庫。

### 1.2 獲取 Anthropic API 金鑰

ArcReel 內建基於 Claude Agent SDK 的 AI 助手，負責劇本創作、智慧對話引導等關鍵環節。

**方式 A：使用 Anthropic 官方 API**

1. 訪問 [Anthropic Console](https://console.anthropic.com/)
2. 註冊賬號並建立 API 金鑰
3. 後續在 Web UI 設定頁配置

**方式 B：使用第三方 Anthropic 相容 API**

如果無法直接訪問 Anthropic API，可在設定頁配置：

- **Base URL** — 填寫中轉服務或相容 API 的地址
- **Model** — 指定使用的模型名稱（如 `claude-sonnet-4-6`）
- 還可分別配置 Haiku / Sonnet / Opus 的預設模型和 Subagent 模型

### 1.3 準備伺服器

**伺服器要求：**

- 作業系統：Linux / MacOS / Windows WSL
- 記憶體：建議 2GB+
- 已安裝 Docker 和 Docker Compose

**安裝 Docker（如未安裝）：**

```bash
# Ubuntu / Debian
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER

# 重新登入後驗證
docker --version
docker compose version
```

## 第二章：部署服務

### 2.1 下載並啟動

#### 方式 A：預設部署（SQLite，推薦入門）

```bash
# 1. 克隆專案
git clone https://github.com/ArcReel/ArcReel.git
cd ArcReel/deploy

# 2. 建立環境變數檔案
cp .env.example .env

# 3. 啟動服務
docker compose up -d
```

#### 方式 B：生產部署（PostgreSQL，推薦正式使用）

```bash
cd ArcReel/deploy/production

# 建立環境變數檔案（需設定 POSTGRES_PASSWORD）
cp .env.example .env

docker compose up -d
```

等待容器啟動完成後，在瀏覽器訪問 **http://你的伺服器IP:1241**

### 2.2 首次配置

1. 使用預設賬號登入（使用者名稱 `admin`，密碼在 `.env` 中透過 `AUTH_PASSWORD` 設定；未設定則首次啟動時自動生成並回寫到 `.env`）
2. 進入 **設定頁**（`/settings`）
3. 配置 **Anthropic API Key**（驅動 AI 助手），支援自定義 Base URL 和模型
4. 配置至少一個圖片/影片**供應商 API Key**（Gemini / 火山方舟 / Grok / OpenAI），或新增自定義供應商
5. 根據需要調整模型選擇、速率限制等引數

> 💡 所有配置項都可以在設定頁修改，無需手動編輯配置檔案。

## 第三章：完整流程

以下步驟在 Web UI 工作臺中完成。

### 3.1 建立專案

1. 在專案列表頁點選「新建專案」
2. 輸入專案名稱（如「我的小說」）
3. 上傳小說文字檔案（.txt 格式）

### 3.2 生成分鏡劇本

在專案工作臺右側開啟 AI 助手面板，透過對話讓助手生成劇本：

- AI 會自動分析小說內容，將其拆分為適合影片的片段
- 每個片段包含畫面描述、出場角色、重要道具/場景（線索）

**稽核點**：檢查劇本結構是否合理，角色和線索是否識別正確。

### 3.3 生成角色設計圖

AI 為每個角色生成設計圖，用於保持後續所有場景中的角色外觀一致。

**稽核點**：檢查角色形象是否符合小說描述，不滿意可重新生成。

### 3.4 生成線索設計圖

AI 為重要道具和場景元素（如信物、特定地點）生成參考圖。

**稽核點**：檢查線索設計是否符合預期。

### 3.5 生成分鏡圖片

AI 根據劇本生成每個場景的靜態圖片，自動引用角色和線索設計圖確保一致性。

**稽核點**：檢查場景構圖、角色一致性、氛圍是否正確。

### 3.6 生成影片片段

分鏡圖片作為起始幀，透過所選影片供應商（Veo 3.1 / Seedance / Grok / Sora 2 等）生成 4-8 秒的動態影片片段。

生成任務進入非同步任務佇列，你可以在任務監控面板實時檢視進度。Image 和 Video 通道獨立併發，RPM 限速確保不超 API 配額。

**稽核點**：預覽每個影片片段，不滿意可單獨重新生成。

### 3.7 合成最終影片

所有片段透過 FFmpeg 拼接，新增轉場效果和背景音樂，輸出最終影片。

預設輸出 **9:16 豎屏**格式，適合釋出到短影片平臺。

## 第四章：進階技巧

### 4.1 版本歷史與回滾

每次重新生成素材時，系統自動儲存歷史版本。在工作臺的時間線檢視中，可以瀏覽歷史版本並一鍵回滾。

### 4.2 控制費用

**檢視費用統計：**

在設定頁可檢視 API 呼叫次數和費用明細。

**減少開支的技巧：**

- 仔細稽核每個階段的輸出，減少返工
- 先生成少量場景測試效果，滿意後再批次生成
- 影片生成使用 Fast 模式可節省約 60% 費用
- 分鏡圖使用 Flash 模型，角色設計圖使用 Pro 模型

### 4.3 專案匯入/匯出

專案支援打包歸檔，方便備份和遷移：

- **匯出**：將整個專案（含所有素材）打包為歸檔檔案
- **匯入**：從歸檔檔案恢復專案

## 第五章：常見問題

### Q: Docker 啟動失敗？

1. 確認 Docker 服務正在執行：`systemctl status docker`
2. 檢查埠 1241 是否被佔用：`ss -tlnp | grep 1241`
3. 檢視容器日誌：`docker compose logs`（在對應的 `deploy/` 或 `deploy/production/` 目錄下執行）

### Q: API 呼叫失敗？

1. 確認設定頁中對應供應商的 API Key 填寫正確
2. Gemini 使用者需確認已啟用付費層級（免費層級不支援圖片/影片生成）
3. 檢查伺服器網路是否可以訪問對應供應商的 API 服務
4. 在供應商控制檯檢視 API 使用量是否超限

### Q: 角色在不同場景中長得不一樣？

1. 確保先生成角色設計圖
2. 檢查角色設計圖質量，不滿意要先重新生成
3. 系統會自動使用角色設計圖作為參考，確保後續場景一致

### Q: 影片生成很慢？

影片生成通常需要 1-3 分鐘/片段，這是正常的。影響因素：

- 影片時長（4 秒 vs 8 秒）
- API 伺服器負載
- 網路狀況

任務佇列支援併發處理，多個影片片段可同時生成。

### Q: 生成中斷了怎麼辦？

任務佇列支援斷點續傳。重新觸發生成時，系統會自動跳過已完成的片段，只處理剩餘部分。
