## ADDED Requirements

### Requirement: 项目大厅必须呈现创作型工作台入口感

项目大厅 MUST 从普通项目列表页升级为创作型工作台入口，使用统一的背景、头部、卡片与状态层级，让用户能快速识别项目、进度与入口操作。

#### Scenario: 用户进入项目大厅
- **WHEN** 用户打开项目大厅页面
- **THEN** 页面 MUST 清晰区分背景层、顶部操作区和项目卡片区，不能呈现为单层深灰列表页

#### Scenario: 项目卡片展示元信息
- **WHEN** 页面渲染项目卡片
- **THEN** 卡片 MUST 清晰区分封面区、标题区、进度区和辅助元信息区，避免主要信息与次要信息全部堆叠在同一视觉层级中

### Requirement: 全局头部必须强化当前上下文与关键操作

工作区全局头部 MUST 清晰表达当前项目、当前阶段、关键状态与高频操作，并通过统一层级规则避免信息拥挤与按钮噪音。

#### Scenario: 用户进入项目工作区
- **WHEN** 全局头部渲染项目名称、阶段和设置等入口
- **THEN** 头部 MUST 让当前项目上下文优先于次要控制项，且状态反馈与操作按钮应使用不同视觉语义

#### Scenario: 头部存在多种操作按钮
- **WHEN** 页面同时显示返回、通知、导出、设置等操作
- **THEN** 主操作、普通操作和状态型入口 MUST 具有稳定的一致性样式，不得通过多个互相竞争的高亮色同时吸引注意

### Requirement: 三栏工作区必须具有明确表面与边界层级

工作区三栏结构 MUST 明确区分资产侧栏、主编辑区域与助手面板的表面层级与边界关系，使用户能快速理解当前聚焦区域。

#### Scenario: 助手面板展开
- **WHEN** 用户展开右侧助手面板
- **THEN** 助手面板 MUST 与主编辑区形成清晰层级差，并保持工作区整体视觉统一，而不是像临时插入的独立深灰块

#### Scenario: 侧栏与主内容同时可见
- **WHEN** 资产侧栏、主内容区和助手面板同时渲染
- **THEN** 三者 MUST 通过边界、表面和密度差异表达不同职责，用户应能快速判断主要编辑焦点所在区域

### Requirement: 壳层刷新不得改变既有信息架构与导航语义

壳层视觉刷新 MUST 保持现有页面导航关系、主要入口位置与业务语义稳定，避免因视觉重构引入新的操作路径变化。

#### Scenario: 用户使用旧路径访问页面
- **WHEN** 用户通过既有导航进入项目大厅、设置页或工作区
- **THEN** 页面路由、入口位置与主要交互语义 MUST 保持兼容，仅升级视觉层级与可读性
