## ADDED Requirements

### Requirement: 统一的创作型工作台视觉 token

系统 MUST 提供一套统一的创作型工作台视觉 token，用于表达背景、面板、浮层、边界、主文字、次文字、主强调色、次强调色、成功、警告与危险状态，并以石板蓝黑、冷白、雾蓝灰为基础色阶。

#### Scenario: 页面读取主题 token
- **WHEN** 任意工作台页面渲染背景、面板、边框或文字样式
- **THEN** 页面 MUST 通过统一 token 获取颜色语义，而不是直接依赖分散的 `gray-*`、`indigo-*` 或孤立色值表达同类角色

#### Scenario: 状态色职责清晰
- **WHEN** 页面渲染成功、警告、危险三类状态反馈
- **THEN** 系统 MUST 分别使用独立的语义状态色表达，不得复用主操作高亮色替代警告或危险状态

### Requirement: 文字层级必须可扫描

系统 MUST 为主标题、正文、辅助说明、禁用信息和状态提示提供明确的文字层级，保证在深色背景下能快速区分主要信息与次要信息。

#### Scenario: 高密度表单中的说明文字
- **WHEN** 页面展示字段标签、辅助说明与当前值提示
- **THEN** 标签、说明与当前值提示 MUST 使用不同层级的文字语义，避免全部落入同一灰度区间

#### Scenario: 大面积深色背景上的正文
- **WHEN** 页面在主背景或 panel 上展示主要正文信息
- **THEN** 正文文字 MUST 使用冷白或等价高对比语义，不能与辅助说明共享同一低对比色阶

### Requirement: 表面层级必须清楚区分

系统 MUST 至少区分页面背景、基础 panel、强调 panel 与浮层四种表面层级，并通过亮度、边界和阴影建立稳定的层次关系。

#### Scenario: 页面包含多个相邻卡片
- **WHEN** 同一页面同时出现摘要块、表单块和警示块
- **THEN** 各块 MUST 通过表面层级或边界强度区分主次，而不是全部使用相同深灰底和相同边框强度

#### Scenario: 浮层与主内容共存
- **WHEN** 页面展示抽屉、悬浮面板或模态内容
- **THEN** 浮层 MUST 与基础 panel 存在清晰层级差，不得与主内容表面完全同层

### Requirement: 高亮反馈必须节制且一致

系统 MUST 将主强调色用于主操作与当前上下文，将次强调色用于 hover、focus 和轻量交互反馈，避免多个饱和高亮色同时争夺视觉优先级。

#### Scenario: 用户聚焦输入框或可点击项
- **WHEN** 用户 hover 或 focus 可交互控件
- **THEN** 控件 MUST 使用一致的高亮反馈语义，且该反馈不能与危险或警告状态混淆

#### Scenario: 页面存在主按钮与普通按钮
- **WHEN** 页面同时展示 primary 和 secondary 操作
- **THEN** primary 操作 MUST 具有更高视觉权重，secondary 操作 MUST 保持克制，不得与 primary 使用相同层级的高亮强度
